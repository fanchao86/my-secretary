# Julie - 私人助理 Skill

私人事务管理助理，管理主人信息、健康档案、知识库、日程任务与提醒事项。

## 架构

SKILL.md 采用四层结构设计：

| 板块 | 职责 |
|------|------|
| **角色** | 身份、职责、性格、身份标识 |
| **铁律** | T1-T12 共12条不可违反的最高优先级规则 |
| **文件** | 目录结构、读写原则、5个文件模板及校验规则 |
| **命令** | 执行顺序、意图路由、14个命令的详细步骤 |

### 铁律一览

| 编号 | 规则 | 核心要点 |
|------|------|----------|
| T1 | 系统时间优先 | 禁止使用模型时间感知，必须调用 `date` 获取系统时间 |
| T2 | 先读后答 | 涉及资料/查询/修改，必须先读文件再回答 |
| T3 | 写入必须验证 | 写入后必须重新读取验证落盘，未验证不得声称完成 |
| T4 | 提醒/定时任务处理 | 定时提醒必须使用 cron 工具，不可用则告知用户 |
| T5 | 禁止虚假承诺 | 禁止声称未执行的操作；意图不明禁止自行归类写入 |
| T6 | 模糊时间不可编造 | "晚上"等模糊时段不可编造具体时间，追问优先于假设 |
| T7 | 相对日期须确认 | "明天"等相对日期须确认具体日期 |
| T8 | 保护数据 | 不擅自清空历史，过期事项标注"已过期"不自动删除 |
| T9 | 建档逐项确认 | 严禁编造用户数据，每项必须用户亲自确认 |
| T10 | 强制检查 | 每次对话开始检查 `data/` 目录，不存在则建档 |
| T11 | 克制提醒 | 不过度提醒，习惯型提醒未达条件不主动提醒 |
| T12 | 写入前校验 | 写入前校验文件结构，不符合先修复模板 |

### 命令一览

| 编号 | 命令 | 说明 |
|------|------|------|
| 1 | 创建主人信息 | 首次使用时建档，逐项收集 |
| 2 | 查询主人信息 | 读取用户档案 |
| 3 | 修改主人信息 | 定位字段修改 |
| 4 | 查询健康档案 | 读取健康档案 |
| 5 | 修改健康档案 | 新增/修改/删除健康条目 |
| 6 | 加入知识库 | 按主题归类追加 |
| 7 | 查找知识库 | 搜索关键词 |
| 8 | 修改或删除知识库 | 定位条目修改/删除 |
| 9 | 加入日程或任务 | 区分日程与任务，支持 cron 定时提醒 |
| 10 | 查询日程或任务 | 按日期/关键词/状态筛选 |
| 11 | 修改或删除日程或任务 | 变更/完成/移除 |
| 12 | 实时巡检 | 每次对话开始自动执行，紧急事项立即提醒 |
| 13 | 每日巡检 | 每日首次对话执行，输出系统状态+今日日程+健康提醒 |
| 14 | 意图不明确 | 询问用户确认意图，不自行归类写入 |

## 安装

将本项目目录放置于 AI 技能加载路径下，确保 AI 能读取到 `SKILL.md` 即可。

```bash
# 示例：克隆到技能目录
git clone <repo-url> <skills-path>/julie
```

无额外依赖，无需构建。

## 使用

### 首次使用

对话中触发关键词（`Julie`、`助理`、`日程`、`待办`、`提醒`等）激活技能。首次使用时 AI 会引导创建主人档案，自动在项目目录下生成 `data/` 目录及数据文件。

### 支持的操作

| 操作 | 示例 |
|------|------|
| 建档 | "Julie，帮我建档，我叫小明" |
| 查询信息 | "我的生日是什么时候" |
| 修改信息 | "把我的昵称改成大明" |
| 健康档案 | "我最近在吃降压药，帮我记一下" |
| 加入知识 | "记住：我喜欢喝美式咖啡" |
| 查找知识 | "我之前说过喜欢什么咖啡" |
| 添加日程 | "明天下午3点有个会议" |
| 查询日程 | "今天有什么安排" |
| 完成任务 | "把明天的会议标记为已完成" |
| 删除事项 | "删掉明天的会议" |

### 定时提醒

添加带具体日期+时间的日程时，Julie 会通过 cron 工具自动创建系统定时任务：

```
cron(action="add", job={
  name: "银行提醒",
  schedule: {kind: "at", at: "2026-04-18T15:00:00+08:00"},
  payload: {kind: "agentTurn", text: "下午3点去银行处理汇款"}
})
```

### 巡检机制

| 巡检类型 | 触发时机 | 输出 |
|----------|----------|------|
| 实时巡检 | 每次对话开始 | 有紧急事项→提醒；无→静默 |
| 每日巡检 | 每日首次对话 | 系统状态 + 今日日程 + 健康提醒 |

### 数据文件

运行时自动生成于 `data/` 目录，不纳入版本控制。

```
data/
├── users.md       主人信息
├── doctor.md      健康档案
├── knowledge.md   知识库
├── todo.md        日程与任务
└── version.json   版本信息
```

## 配置

所有行为规则和文件模板定义在 `SKILL.md` 中，修改该文件即可调整助理行为。
